The Dcrypt algorithm is an algorithm that uses SHA256 internally. What makes this algorithm FPGA/ASIC-resistant
is the way it was designed.

  • It is unknown beforehand how many internal SHA256 hashes will be calculated
  • It is unknown beforehand how much system memory will be required
  • Large amounts of memory IO are required
  • Each step of the Dcrypt algorithm requires the result of the previous step, making parallelization difficult/impossible
  • The final step of Dcrypt is to SHA256 hash a large generated piece of data of variable size
The Dcrypt CPU Miner source code can be found on the slimcoin-project github repository
